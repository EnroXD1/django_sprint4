# Django Sprint 4 — Блогикум

## Описание
**Блогикум** — учебный Django-проект с публикациями, категориями, профилями пользователей и системой комментариев.

В проекте реализованы:
- главная страница со списком публикаций;
- страницы категорий и профилей авторов;
- регистрация, вход, выход, смена и восстановление пароля;
- создание, редактирование и удаление постов;
- загрузка изображений к публикациям;
- комментарии к постам с возможностью редактирования и удаления;
- пагинация;
- кастомные страницы ошибок 403, 404 и 500;
- отправка писем через файловый backend в директорию `sent_emails/`.

Проект выполнен на **Django 3.2** и использует **SQLite** по умолчанию.

## Установка и запуск

### 1. Клонирование репозитория
```bash
git clone https://github.com/EnroXD1/django_sprint4.git
cd django_sprint4
```

### 2. Создание и активация виртуального окружения
**Windows:**
```bash
py -3.10 -m venv venv
venv\Scripts\activate
```

**Linux / macOS:**
```bash
python3.10 -m venv venv
source venv/bin/activate
```

### 3. Установка зависимостей
```bash
python -m pip install --upgrade pip setuptools wheel
pip install -r requirements.txt
```

### 4. Переход в директорию проекта
```bash
cd blogicum
```

### 5. Применение миграций
```bash
python manage.py migrate
```

### 6. Загрузка тестовых данных из фикстуры
Если в проекте есть файл `db.json`, его можно загрузить так:
```bash
python manage.py loaddata ../db.json
```

### 7. Запуск сервера разработки
```bash
python manage.py runserver
```

После запуска сайт будет доступен по адресу:
```text
http://127.0.0.1:8000/
```

## Основные возможности проекта

### Публикации
- вывод постов на главной странице;
- отдельная страница поста;
- отложенные публикации по дате;
- отображение только опубликованных постов для обычных пользователей;
- автор видит свои посты, включая отложенные и снятые с публикации;
- изображения у публикаций.

### Пользователи
- регистрация нового пользователя;
- авторизация и выход из аккаунта;
- редактирование профиля;
- смена и восстановление пароля;
- страница профиля с публикациями автора.

### Комментарии
- создание комментариев только авторизованными пользователями;
- редактирование и удаление только собственных комментариев;
- сортировка комментариев от старых к новым;
- вывод количества комментариев у постов.

### Дополнительно
- пагинация по 10 записей;
- кастомные страницы ошибок;
- статические страницы `about/` и `rules/`;
- сохранение писем в папку `sent_emails/`.

## Полезные команды

### Создать администратора
```bash
python manage.py createsuperuser
```

### Сбросить базу и загрузить фикстуры заново
**Windows:**
```bash
del db.sqlite3
python manage.py migrate
python manage.py loaddata ../db.json
```

**Linux / macOS:**
```bash
rm db.sqlite3
python manage.py migrate
python manage.py loaddata ../db.json
```

## Важное замечание по фикстурам
В проекте используется **кастомная модель пользователя**:
```python
AUTH_USER_MODEL = 'users.User'
```
Поэтому в фикстурах пользователи должны быть указаны как:
```json
"model": "users.user"
```
а не `auth.user`.
